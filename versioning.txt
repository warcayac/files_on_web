For version 1.9.0.2 (in Flutter it'd be 1.9.0+2):

  1: Major revision (new UI, lots of new features, conceptual change, etc.)

  9: Minor revision (maybe a change to a search box, 1 feature added, collection of bug fixes)

  0: Bug fix release

  2: Build number (if used)—that's why you see the .NET framework using something like 2.0.4.2709

You won't find a lot of apps going down to four levels, 3 is usually sufficient.

VERSION NAME = 1.9.0 (for public)
VERSION CODE = 2 (internal)

* versionCode — A positive integer used as an internal version number. This number is used only to determine whether one version is more recent than another, with higher numbers indicating more recent versions. 

En Flutter es suficiente con especificar en el campo VERSION del archivo pubspec.yaml
Pero esto puede ser omitido si al generar el aplicativo a publicar se corre:

flutter build apk --build-name = 1.0.2 --build-number = 3
